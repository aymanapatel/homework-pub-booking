# answers/

This is where you write your reasoning for each exercise. Each `ex{N}_*.md`
file is a prompt template; you fill in the answers.

**Rules:**

- The grader expects every file to exist, even if an answer is short.
- Placeholder text (`TODO`, `<your answer here>`, or empty) is scored zero.
- Cite specific ticket IDs, trace lines, or session directory paths where
  relevant. The grader cross-references what you write against the
  `sessions/` artifacts committed to your repo at submission time.
- Keep answers to the word-count range specified in each template. Shorter
  than the floor or longer than the ceiling both lose marks.

Do NOT modify the headings or the "prompt" sections — those are how the
grader locates your answers within the file. Write under the "Your answer"
headings.
